# watch.sideby.me Documentation

This directory documents how the watch.sideby.me frontend is structured and how to work on it. It covers the client-side app only — for the Socket.IO backend, see [`sync.sideby.me/docs/`](https://github.com/sideby-me/sync.sideby.me/docs/).

## Contents

- `architecture.md` — How the frontend is structured and how it connects to `sync.sideby.me`
- `file-structure.md` — Directory layout and what lives where
- `local-development.md` — Running the app locally
- `contributing.md` — Where to put new code and how to work with sockets
- `code-review.md` — What reviewers check for
- `naming-and-style-guide.md` — Component, hook, and event naming conventions
- `patterns-and-best-practices.md` — Frontend patterns used in this codebase
- `errors-and-messaging.md` — How to handle error events from `sync.sideby.me`
- `logging-and-observability.md` — Client-side logging helpers
- `performance.md` — Client-side performance hotspots
- `real-time-flows.md` — How socket events are wired on the client
- `troubleshooting.md` — Common issues and where to look
- `deployment-and-operations.md` — Building and running in production
- `ui-copy-and-language.md` — Voice, tone, and copy guidelines
